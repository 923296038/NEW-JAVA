1.背景介绍

数组和集合

数组：可以存储基本数据类型和对象，数组的长度固定，不适合在对象数量未知的情况下使用。

集合：只能存储对象，对象类型可以不一样，长度可变，可在多数情况下使用。

2.知识剖析

泛型

泛型E像一个占位符一样表示“未知的某个数据类型”，我们在真正调用的时候传入这个“数据类型”。

Controllect接口

Collection接口的两个子接口是List、Set接口，由于List、Set是Collection的子接口，意味着所有List、Set的实现类都有上面的方法。

Map接口

Map就是用来存储“键(key)-值(value) 对”的。 Map类中存储的“键值对”通过键来标识，所以“键对象”不能重复。

List、Set、Map接口的比较

List（有序、可重复、允许存在多个null元素）

Set（无序、不能重复、只允许存在一个null元素）

Map（键值对、键唯一、值不唯一、只允许存在一个Key为null元素 ）

list接口

有序：List中每个元素都有索引标记（在List中的位置）。

可重复：List允许加入重复的元素。

ArrayList底层是用动态数组实现的存储。 特点：查询效率高，增删效率低，线程不安全。

LinkedList底层用双向链表实现的存储。特点：查询效率低，增删效率高，线程不安全。

set接口

无序：Set中的元素没有索引，我们只能遍历查找;

不可重复：不允许加入重复的元素。

Set接口继承自Collection，Set接口中没有新增方法，方法和Collection保持完全一致。前面的List学习的方法，在Set中仍然适用。

Set的实现类有：HashSet、TreeSet、LinkedHashSet、EnumSet等，我们一般使用HashSet。

Map接口

Map就是用来存储“键(key)-值(value) 对”的。 Map类中存储的“键值对”通过键来标识，所以“键对象”不能重复。

键值对，无序。

键唯一、值不唯一

只允许存在一个Key为null元素

Map 接口的实现类有HashMap、TreeMap、HashTable、Properties等。我们一般使用HashMap。

1.为什么要声明为List 而不是ArrayList

设计模式中有：“代码尽量依赖于抽象，不依赖于具体”。

通过list来操作集合。代码编写后发现集合使用的不准确，应该使用LinkedList，那么只要修改一行代码List list = new LinkedList();就可以，这行以后的代码不需要修改，因为List接口保证了调用的都是接口中的方法，而ArrayList与LinkedList都实现了List接口。

而如果当时用ArrayList list = new ArrayList()这种形式的话，那么list访问到的就可能是ArrayList里独有的方法而非List接口中的方法。这样替换成LinkedList的时候就有可能需要修改很多的代码。
